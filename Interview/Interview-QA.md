# General Interview Preparation – Q&A

## � TECHNICAL INTERVIEW PREPARATION

### Before the Interview

**Q:** How to prepare for a technical interview?
**A:** Review frameworks and concepts in this playbook. Practice coding problems on LeetCode. Build projects. Understand system design basics. Review your own projects. Get 8 hours sleep before. Calm mind, focus.

**Q:** What to do 1 day before interview?
**A:** Light review (don't overload brain). Check interview format, duration, tools (Zoom, CodePen). Test your setup (camera, microphone, internet). Prepare environment (quiet, well-lit). Lay out professional clothes.

**Q:** How to handle nervousness?
**A:** Deep breathing exercises. Practice mock interviews. Remember: interviewers want you to succeed. Take 1-2 seconds before answering. Think out loud. Ask clarifications if unsure.

**Q:** What to do during interview?
**A:** Greet professionally. Listen carefully. Ask clarifying questions. Explain your approach before coding. Communicate while solving. Ask for feedback. Thank interviewers. Follow up with thank you email.

### Interview Format Knowledge

**Q:** What are common interview formats?
**A:** Behavioral (experience, past projects). Technical (coding, system design, trivia). Take-home assignment (build feature, submit code). Pair programming (collaborative coding). Vary by company and role level.

**Q:** What is behavioral interview?
**A:** Questions about past experiences. "Tell about a time you...". Stories with context, action, result. Shows problem-solving, teamwork, learning. Practice STAR method (Situation, Task, Action, Result).

**Q:** What is technical interview?
**A:** Coding problems, system design, technical trivia. Assess problem-solving, knowledge, communication. LeetCode-style problems common. Whiteboard or online editor. Explain approach clearly.

**Q:** What is system design interview?
**A:** Design large-scale system (Twitter, YouTube, Uber). Scalability, availability, consistency. Databases, caching, queues, load balancing. Discuss trade-offs. Shows architectural thinking.

**Q:** What is take-home assignment?
**A:** Build feature/project at home. Submit code within timeframe. Evaluated on code quality, completeness, testing. Shows real-world work style. Take ~2-4 hours, not more.

### Problem-Solving Approach

**Q:** How to approach coding problem?
**A:** 1) Read problem carefully. 2) Ask clarifying questions. 3) Think of approach (brute force first). 4) Optimize (time/space). 5) Code. 6) Test edge cases. 7) Refactor. Never rush into coding.

**Q:** What to ask interviewers?
**A:** Problem constraints (input size, time limit). Edge cases (empty, negative, duplicates). Output format. Follow-up optimization needed. Show you think thoroughly before coding.

**Q:** What are common edge cases?
**A:** Empty input ([], ""). Single element. Duplicates. Negative numbers. Very large numbers. Null/undefined. Special characters. Consider all scenarios.

**Q:** How to optimize for time complexity?
**A:** Identify current complexity. Look for redundant operations. Use data structures (hash map, set). Reduce nested loops. Divide and conquer (recursion). Binary search instead of linear. Trade space for time.

**Q:** How to optimize for space complexity?
**A:** Identify current usage. Avoid unnecessary variables. In-place modifications if possible. Streaming data vs loading all. Memory trade-offs with time. Understand O(1) vs O(n) space.

## � BEHAVIORAL & SOFT SKILLS

**Q:** What is STAR method for behavioral answers?
**A:** Situation - Set context. Task - Your responsibility. Action - What you did. Result - Outcome/learning. Structure: 1-2 min story. Shows competency, takes initiative.

**Q:** What are common behavioral questions?
**A:** "Tell about challenging project." "Conflict with teammate, resolution." "Learned from failure." "Proud achievement." "Leadership example." "Handled pressure." Prepare 5-7 stories covering different competencies.

**Q:** How to answer "Why us?"
**A:** Research company. Know products, values, mission. Genuine interest, not just salary. Match your goals with company. Reference specific things you admire. Shows thoughtfulness.

**Q:** How to answer "Why are you leaving?"
**A:** Positive framing (seeking growth, new challenges). Avoid criticizing previous employer. Focus on opportunity, learning, company fit. Not about money/benefits primarily.

**Q:** How to answer "What's your weakness?"
**A:** Real but improvable weakness. Show self-awareness. Discuss improvement efforts. Not critical to job. Example: "I used to multitask too much, now I focus on one task fully." Shows growth.

**Q:** What to ask interviewer?
**A:** "What does typical day look like?" "Team composition?" "What success looks like?" "Company culture?" "Growth opportunities?" Shows interest, not just taking a job. Ask 3-4 questions.

## � TECHNICAL CONCEPTS SUMMARY

### Frontend Basics

- HTML: structure, semantic tags, accessibility
- CSS: selectors, box model, flexbox, grid, responsive design
- JavaScript: data types, functions, closures, async/await, DOM manipulation
- React/Vue/Angular: components, state management, lifecycle, hooks

### Common Data Structures

- Array (O(1) access, O(n) insertion)
- Object/Hash Map (O(1) lookup, space overhead)
- Linked List (O(n) access, O(1) insertion if pointer known)
- Tree (O(log n) balanced, hierarchical data)
- Graph (relationships, algorithms: BFS, DFS, Dijkstra)

### Algorithm Complexity

- O(1): constant
- O(log n): binary search
- O(n): single loop
- O(n log n): sorting (merge, quick)
- O(n²): nested loops
- O(2ⁿ): exponential (avoid)

### Database Concepts

- SQL: SELECT, JOIN, GROUP BY, indexing
- NoSQL: documents, flexibility, scalability
- Query optimization, indexing importance
- Transactions, acid properties

### System Design

- Scalability: load balancing, caching, database sharding
- Availability: redundancy, failover, monitoring
- Consistency: eventual consistency, CAP theorem
- Common patterns: MVC, singleton, observer

## � INDIAN INTERVIEW CONTEXT

**Q:** What are Indian companies interviewing for?
**A:** FAANG (Facebook, Apple, Amazon, Netflix, Google): algorithm-heavy, system design. Startups: full-stack, quick learning. Fintech: security, scalability, transactions. Design systems: component libraries, design tokens.

**Q:** How Indian fintech interviews differ?
**A:** Payment systems, security, GDPR/local compliance. PCI-DSS standards. Fraud detection algorithms. Real-time settlement. Transaction throughput. Risk assessment. Regulatory knowledge.

**Q:** What do Indian startups value?
**A:** Speed of development. Wearing multiple hats. Problem-solving, resourcefulness. Learning new tech quickly. Initiative, ownership. Communication (English + Hindi often). Startup experience advantage.

**Q:** Salary negotiation tips (India)?
**A:** Research market rates on Glassdoor, Ambition Box. Know your worth. Discuss at end after offer. Include bonus, stocks, perks. Don't anchor too high/low. Negotiate professionally. Get offer in writing.

**Q:** Common Indian interview challenges?
**A:** English communication (practice pronunciation). Time zone differences (be on time). Internet stability (have backup internet). Formal culture (Tier 1 fintech/IT). Competitive pool (many candidates).

## � POST-INTERVIEW

**Q:** What to do after interview?
**A:** Send thank you email within 24 hours. Reference specific conversation points. Reaffirm interest. Keep professional, concise. Wait for recruiter feedback. Follow-up email if no response in 1 week.

**Q:** How to handle rejection?
**A:** Ask for feedback (constructive). Learn from interview. Reflect on weak areas. Improve and apply again later. Rejection is normal, everyone faces it. Focus on next opportunity.

**Q:** How to handle multiple offers?
**A:** List pros/cons for each. Company growth potential. Team quality. Learning opportunities. Salary/benefits. Work-life balance. Career path. Gut feeling matters. Choose based on goals.

**Q:** What after getting offer?
**A:** Negotiate if needed. Get offer letter. Resign properly (2 weeks notice typical). Transition knowledge. Maintain relationships. Start date preparation. Onboarding docs review.
